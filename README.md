# PromptFly
#### Video Demo: https://youtu.be/FVf3Vpdl1nk
#### Description:

PromptFly is a Chrome extension I built for my CS50 final project. The main goal was to stop the annoying process of typing out the same complex prompts over and over again into ChatGPT, search boxes, or basically any text field you use daily. Instead of copying and pasting from a separate notes app, you just save your favorite text snippets once into the extension, and you can instantly recall them anytime using a simple "//command" shortcut right while you are typing on any website. 

How it works in practice is pretty straightforward. Whenever you are typing in a normal text input or a textarea field on a website, you just type a double slash "//" followed by at least two letters of a command you saved earlier. For example, if you saved a command called "debug", you would just type "//de". As soon as you do that, a small custom autocomplete window pops up directly underneath the text field you are typing in, showing you all the commands that match what you've typed so far. You can then use your keyboard arrow keys to go up and down the list, and just hit Enter (or naturally click on the suggestion with your mouse) and it immediatly replaces the "//de" text with your full, long saved prompt.

The architecture of the project is broken down into four main files. 

The biggest file is "content.js". This is the main script that runs silently in the background of every single web page you visit. It attaches event listeners to the page to listen for the double slash trigger. I originally tried using just a single slash as the trigger, but that turned out to be a terrible design choice because single slashes are used constantly when typing out URLs or directory paths. It got incredibly annoying when the popup kept triggering by mistake every time I typed a web address. Changing it to a double slash completely fixed those false positives. For the same reason, I made a strict rule that the popup will not even show up until you have typed at least two characters after the slashes. 

Another big design decision in the content script was implementing "lazy DOM injection". I didn't want the extension to slow down the user's browser, so the HTML elements and the CSS styling for the popup box are not actually added to the webpage code until the absolute first time you trigger a command. Until you type those slashes, the extension sits quietly in the background using barely any computer memory. I also had to figure out a trick where clicking a dropdown item uses the "mousedown" event coupled with "preventDefault". If I used a normal click event, the text box would lose its focus before the script had a chance to insert the text, breaking the whole process.

The second core file is "popup.js". This script controls the logic inside the actual extension window that drops down when you click the PromptFly icon in your browser toolbar. This is the control center where you add, edit, or delete your library of saved commands. To make it feel responsive, I added live validation in the javascript. If you try to create a command name that is under 2 characters long, the input box boundary turns a warning color immediately. And if you still try to save it, the placeholder text temporarily switches to a warning message instead of throwing an ugly, intrusive browser alert popup. 

The third file is "popup.html" which simply contains the structure for that popup window. I made it a point to keep the styling entirely self-contained by writing pure custom CSS. I didn't use any heavy external styling libraries like Bootstrap or Tailwind, which kept the project extremely lightweight.

The final file is "manifest.json" which is just the standard configuration file required for all modern Chrome extensions using Manifest V3. It requests the storage permission so it can save your commands locally, and the scripting and activeTab permissions so it can correctly inject the content script into the page you are viewing. I set the host permissions to all URLs, which guarantees the shortcut feature will work seamlessly no matter what website you are currently browsing.

I thought about adding a background service worker script, which a lot of tutorials recommend, but I realized my extension didn't actually need one to function perfectly. The saved commands just load into the content script's memory once when you open a page, and they stay automatically synced with chrome.storage wether the popup menu is open or closed, so a background script would have just been bloat.
